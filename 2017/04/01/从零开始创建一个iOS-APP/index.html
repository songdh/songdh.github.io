从零开始创建一个iOS APP

每次创建app的时候，有很多地方需要修改，在此记录一下，以防有疏漏

一、创建工程

打开XCode，点击菜单File->File->New->Project->Signle View Application->Next

在此界面输入app的一些信息:

  1. Product Name:产品名称，即在手机上显示的APP的名称，同时也是项目的工程名字。一般情况，工程名字都是英文命名，而app显示名字是中文的，所以此处可以用英文来命名，用做工程名字，app显示的名字我们后续再去修改。
  2. Team:没有用过
  3. Organization Name:组织名称，就是此项目版权主体是谁，是公司就写公司名字，是个人就写个人名字。
    这个名字写好后，在项目中创建的任何源文件，都会在文件头部带有这个版权信息的。
    Copyright © 2017年 schoollife. All rights reserved.
  4. Organization Identifier:和Bundle Identifier:每一个组织下面可能会有多个产品。每一个产品需要有一个惟一的ID来表示，即Bundle Identifier,此ID非常重要，一旦发布就不能够再修改。比如com.songdh.office,com.songdh.school,这是两个ID,那么com.songdh就表示的是组织ID，office和school表示此组织下的两个应用。
  5. Language:选用swift或者Objectivie-C来开发
  6. Devices:iPhone(仅iPhone)/iPad(仅iPad)/Universal(前两者均包含)
  7. Use Core Data:由于Core Data框架使用起来比较复杂，如果需要用到，那么勾选这一项，XCode会自动为我们添加一个框架方便开发使用CoreData功能.
  8. Include Unit Tests:添加单元测试框架
  9. Include UI Tests:添加UI测试框架

按照引导步骤创建完工程，然后在模拟器上运行一下，此时只是显示一个白屏。

二、修改项目配置文件

Info.plist文件是工程的配置文件，此文件非常重要，APP的很多配置需要在这个文件中完成

  1. 修改APP显示名称。前面说过，APP显示名称是可以修改的。在文件中我们看到bundle name 的value值为$(PRODUCT_NAME),我们可以直接把$(PRODUCT_NAME)改为我们要显示的名字，也可以去修改设置项。我们选择后一种方式：在左侧点击工程名字，选择heibaiOffice的TARGET，在右侧选择Build Setting,在旁边的搜索框中输入product name,可以看到现在显示的name为heibaiOffice,将其改为黑白办公室。好了再运行下项目，看名字是不是改了。
  2. ATS设置。应用默认的网络请求协议是HTTPS，如果接口还不支持的话，可以修改为支持HTTP.在plist文件中，鼠标放在Information Property List上，其右侧会出现一个加号，点击加号，输入App Transport Security Settings,注意大小写。可以看到它是一个字典格式的。点击其左侧的剪头使剪头朝下展开，再点击其右侧的加号，这是会出现一个下来提示菜单，我们选择Allow Arbitrary Loads，将其值设置为YES。其他值的作用请自行查找。
  3. 设置应用的方向。如图所示
    a. Device Orientaion表示APP支持的屏幕旋转方向
    b. Portrait:正常方向
    c. Upside Down:上下颠倒
    d. Landscape Left:向左横屏
    e. Landscape Right:向右横屏
    在此我们把其他选项都去掉，只选Portrait，即不支持屏幕旋转。
  4. APP最低支持版本。如上图所示，Deployment Target显示的是最新的10.3版本SDK，点向下箭头，选择支持的最低版本，这里选择8.0
  5. 修改启动图片和APP Icon。查看工程设置的General选项，如下图。工程默认是使用LaunchScreen来加载启动图的。我们在工程中找到LaunchScreen.storyboard文件，将其删除。然后在Info.plist中找到Launch screen interface file base name这一项，将其删除。将图中Launch Screen File项删除。点击Use Asset Catalog->Migrate。 在工程中点击Assets.xcassets，可以看到目录下有两个文件。下载图片，将两个目录LaunchImage.launchimage和AppIcon.appiconset替换拷贝到Assets.xcassets中。然后运行项目查看。若是没有改变，可以将APP删除再运行一次。
